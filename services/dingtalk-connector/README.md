# 军团作战助手 · 钉钉 Stream 连接器

连接器只负责两件事：

1. 通过钉钉 Stream 接收单聊/群聊 @ 消息，并把钉钉身份和问题交给 Supabase Edge Function。
2. 从通知发件箱领取已经过权限计算的 AI 行动摘要，主动发给绑定的销售。

数据库高权限密钥和模型密钥都不放在连接器中。连接器只持有钉钉应用凭证和一枚独立的连接口令。

## 本地试点

1. 复制 `.env.example` 为 `.env`，在本机填写真实值。不要把 `.env` 发到聊天或提交到 Git。
2. 安装依赖：`npm ci`
3. 启动：`npm run start:env`
4. 看到“钉钉 Stream 已连接”后，在钉钉里单聊“军团作战助手”。
5. 首次消息会生成待绑定身份。管理员在网页“系统管理 → 钉钉身份”确认后，再次提问即可读取对应权限范围的数据。

主动通知在试点阶段默认关闭。只有设置
`DINGTALK_NOTIFICATIONS_ENABLED=true`，并填写
`DINGTALK_NOTIFICATION_STAFF_ALLOWLIST` 后才会发送。

领导日报和周报采用 Supabase 云端定时直发，不经过本机通知轮询，因此电脑合盖、
关机或 Stream 连接器离线不会影响已经绑定领导的定时报告。Stream 连接器仍用于
接收领导和销售主动发给机器人的对话。

管理员可以预先创建一次性身份认领。被预授权人员下一次向机器人发送文本消息时，
服务端会使用钉钉消息中的真实 `staffId` 完成绑定；昵称不匹配、多人匹配或授权过期
都不会自动绑定。

健康检查地址：`http://127.0.0.1:8787/health`

## macOS 登录后自动启动

本地试点可使用 `scripts/start-macos-launchagent.sh` 作为 macOS LaunchAgent
的启动入口。脚本从登录钥匙串读取以下三项，不把密钥写入启动配置：

- `battlemap-dingtalk-client-id`
- `battlemap-dingtalk-client-secret`
- `battlemap-dingtalk-connector-token`

LaunchAgent 需要另外提供 `DINGTALK_AGENT_ENDPOINT` 等非密钥环境变量，并设置
`RunAtLoad` 和 `KeepAlive`。电脑锁屏时服务继续运行；电脑合盖睡眠期间连接会暂停，
开盖联网后由钉钉 SDK 自动重连。关机或重启后，用户重新登录时服务自动启动。

## 对话确认写入

绑定用户可以直接向机器人描述项目进展或当天工作。Agent 只生成待确认清单，不会在
第一条消息中修改业务数据：

1. 销售只能更新本人负责的项目，并录入本人的日报。
2. 管理员可以按日期一次发送多名销售的日报，Agent 会分别匹配销售和项目。
3. 项目或销售无法唯一匹配时不写入，并要求用户补充。
4. 用户回复“确认更新”后，服务端才会原子写入项目、项目活动和日报。
5. 用户回复“取消更新”则放弃；待确认内容 24 小时后自动失效。
6. 重复确认不会重复写入，项目在等待期间被其他人修改时会要求重新生成。

待确认项会同步显示在网页“AI 认知行动 → 待我确认”中。所有实际写入保留原始消息、
操作账号以及修改前后值。

## 单独验证主动通知

在已填写 `.env` 的情况下执行：

`node --env-file=.env src/proactive-check.mjs --staff-id=测试人员staffId`

该命令只验证钉钉发送通道，不读取或修改销售数据。
